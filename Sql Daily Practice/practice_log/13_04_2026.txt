## Learnings

### SQL Basics
- GROUP BY requires proper column selection
- ORDER BY defaults to ASC
- DISTINCT removes duplicate rows
- Subqueries must return correct structure (single vs multiple rows)

---

### JOIN Concepts
- INNER JOIN returns only matching records
- LEFT JOIN keeps left table fully
- RIGHT JOIN keeps right table fully
- JOIN direction depends on question requirement
- NULL in JOIN means no match found

---

### Aggregation with JOIN
- COUNT(e.id) avoids counting NULL rows
- GROUP BY works after JOIN
- Selecting correct LEFT table is critical for correct output

---

### Common Mistakes Fixed
- Confusing JOIN direction
- Using `!=` instead of `IS NULL`
- Using wrong table as base in LEFT JOIN
- Misunderstanding NULL behavior in joins
- Forgetting GROUP BY rules
